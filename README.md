# coding-agent-harness

> 코딩 에이전트(LLM)가 **사람과 협업하는 방식**을 정의한 하네스.
> AI가 계획을 주도하고, 사람은 승인·검증한다.

코딩 에이전트에게 "어떻게 일할지"를 알려주는 규칙 모음입니다. 핵심은 [`CLAUDE.md`](./CLAUDE.md) 한 파일 — 협업 원칙, 작업 수칙, 문서화, 기록·트래킹 규칙이 담겨 있습니다. 새 프로젝트의 `CLAUDE.md`에 그대로 옮기거나, 프로젝트별 지침과 병합해 쓸 수 있습니다.

## 왜 만들었나

LLM 코딩 에이전트는 빠르지만 흔한 실패가 있습니다 — 시키지도 않은 걸 과설계하고, 멋대로 인접 코드를 건드리고, 검증 없이 "완료"라 하고, 작업 흔적을 남기지 않아 다음 세션이 맥락을 잃습니다.

이 하네스는 그 실패를 **사람이 매번 잔소리하지 않아도** 줄어들게 하는 게 목적입니다. 동시에 사람을 매 단계 승인 버튼이나 누르는 존재로 만들지 않습니다 — **AI가 주도하고, 사람은 계획과 결과 두 지점에서만 검증**합니다.

## 핵심 철학

- **AI가 계획을 주도하고, 사람은 승인·검증한다.** AI가 계획을 먼저 제시하고, 주도적으로 진행하며, 결과를 사실대로 보고한다. 사람의 게이트는 *계획 승인*과 *결과 검증* 두 지점이 기본 — 매 단계 승인을 구하지 않는다.
- **요구자도 사람이다 — 요구 자체도 틀릴 수 있다.** AI가 틀릴 수 있는 만큼 사람의 요구도 틀릴 수 있다. 단어가 의도와 어긋나 보이면 되묻는다.
- **검증까지가 완료다.** 작성·수정 후 실제로 동작하는지 직접 돌려 확인한 뒤에야 "완료".
- **수술하듯 바꾼다.** 요청에 직결된 것만 건드리고, 내 변경이 만든 흔적만 정리한다.
- **최소한으로 만든다.** 요청하지 않은 기능·추상화·유연성은 넣지 않는다.

## 구성

[`CLAUDE.md`](./CLAUDE.md) 는 네 섹션으로 이뤄져 있습니다.

| 섹션 | 다루는 것 |
|---|---|
| **협업 원칙** | AI 주도/사람 승인·검증, 계획은 가설, 요구자도 틀릴 수 있음 |
| **작업 수칙** | 검증까지가 완료, 수술하듯 변경, 운영·인프라 재현성, 최소한 |
| **문서화** | 도메인 문서, 외부 노출용 도메인 문서, ADR(폐기한 길과 이유) |
| **기록·트래킹** | git / `PROGRESS.md`(현재 상태) / `PM-TASKS.md`(보고 업로드용) |

## 사용법

1. 이 저장소의 [`CLAUDE.md`](./CLAUDE.md) 를 프로젝트 루트로 복사합니다.
2. 프로젝트 고유의 지침(빌드 명령, 스택, 컨벤션 등)을 같은 파일에 덧붙입니다.
3. Claude Code 등 코딩 에이전트가 자동으로 읽어 협업 규칙으로 사용합니다.

> 이 규칙들은 캡션이 아니라 **방향**입니다. 사소한 작업에는 판단으로 가볍게 적용하세요.
